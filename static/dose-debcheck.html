Debian Quality Assurance

Dose-Debcheck: Not Installable Packages

Currently checked scenarios

  • unstable, main only
  • unstable contrib and non-free, against main background
  • testing, main only
  • testing contrib and non-free, against main background

What this means

We call a package installable if all its (recursive) dependencies can be satisfied without violating any conflicts. Alternative dependencies and virtual packages are taken into account. More precisely, a package p is called installable in a package respository D if there there is a subset S of D containing p, and such that for each package in S

  • every dependency relation is satisfied by some package in S
  • and it is not in conflict with any package in S
Packages from a background can be used to satisfy dependencies, but we are not interested in background packages per se.

The set of not installable packages is computed by the dose-debcheck tool available in the dose-distcheck package, the successor of the previously used edos-debcheck tool.
